# Контракты взаимодействия с сервером

Информация по каждой шашке:
    положение на доске x=[0 .. 7], y=[0 .. 7]
    тип шашки: черная, черная дамка, белая, белая дамка
    список доступных для шашки ходов: list(tuple(x: int, y: int))

Далее под состоянием поля будем подразумевать информацию о состоянии всех шашек оставшихся в игре.

Взаимодействие сервера с клиентами:
    В начале игры сервер отправялет клиентам информацию по всем шашкам и информацию о том, за какой цвет играет каждый из
    клиентов. Для клиента который должен ходить следующим сервер в рамках информации о его шашках отправляет список доступных
    ходов. Из этого перечня клиент выбирает один из ходов и отправляет об этом информацию на сервер. Сервер обновляет состояние
    поля у себя и отправляет второму клиенту информацию о совершенном противником ходе и новое состояние поля. Стоит отметить,
    что один и тот же игрок можем ходить несколько раз подряд, если он должен съесть несколько шашек. Когда сервер отправляет
    возможные ходы для шашки, он указывает только те ходы, при выполнении которых съедается не больше одной шашки противника.

    Итого:
        1. Сервер отправляет состояние поля

        Пока не останутся шашки только одного цвета:
            2. Клиент у шашек которого есть доступные ходы совершает ход и отправляет его серверу
            3. Сервер обновляет у себя состояние поля
            4. Отправляет противнику сделанный ход
            5. Отправляет обоим клиентам новое состояние поля
